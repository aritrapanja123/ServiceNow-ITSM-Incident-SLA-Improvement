# Project Charter

## Project Title

ServiceNow ITSM Incident and SLA Improvement Analytics

## Fictional Company

NorthBridge Retail Group is a fictional UK retail organisation operating over 250 stores, supported by a central IT service desk. The company uses ServiceNow to manage incidents related to POS systems, payment terminals, Microsoft 365, network issues, SAP/ERP access, warehouse scanners and reporting systems.

## Business Problem

NorthBridge Retail Group has experienced an increase in IT incident volumes and SLA breaches across retail stores, warehouses and head-office operations. Service managers currently lack clear visibility into why tickets breach SLA, which assignment groups are overloaded, which incident categories create recurring issues and which incidents should be escalated before they breach.

The current incident-management process is highly manual, with inconsistent categorisation, delayed reassignment, late escalation and limited recurring-incident detection.

## Central Business Question

How can NorthBridge Retail Group reduce SLA breaches, improve incident resolution speed and automate escalation for recurring and high-risk IT incidents?

## Project Objectives

1. Analyse incident volume, SLA compliance and resolution performance.
2. Identify the main drivers of SLA breaches.
3. Detect recurring incidents by category, configuration item, location and assignment group.
4. Analyse reassignment delays and overloaded support teams.
5. Design an improved incident-management and escalation workflow.
6. Build a Power BI dashboard for IT service managers.
7. Create business requirements, user stories, acceptance criteria and UAT test cases.
8. Estimate potential operational and financial benefits.

## Scope

### In Scope

- Incident ticket analysis
- SLA breach analysis
- First-response and resolution-time analysis
- Assignment group performance
- Reassignment delay analysis
- Recurring incident identification
- Escalation-rule design
- Power BI dashboard
- AS-IS and TO-BE process maps
- Requirements documentation
- UAT test cases
- Business case

### Out of Scope

- Real ServiceNow API integration
- Real employer or customer data
- ServiceNow configuration changes
- Change-management tickets
- Request-fulfilment tickets
- Full problem-management implementation
- Machine-learning prediction model

## Key Stakeholders

| Stakeholder | Role |
|---|---|
| IT Service Manager | Owns SLA performance and service improvement |
| Service Desk Team Lead | Manages ticket triage and analyst workload |
| Assignment Group Managers | Own specialist-team resolution performance |
| Store Operations Manager | Represents retail-store users |
| IT Reporting Analyst | Owns dashboard and KPI reporting |
| Problem Manager | Reviews recurring incidents |
| Business Analyst | Defines requirements and validates solution |

## Success Criteria

The project will be successful if it can:

- Identify top SLA breach categories.
- Show which assignment groups have the highest breach rates.
- Highlight incidents at risk before SLA breach.
- Detect recurring incidents suitable for problem-management review.
- Reduce manual reporting through dashboard automation.
- Provide clear requirements and UAT test cases.
- Recommend a practical TO-BE escalation process.
